# Deprecated T-SQL Syntax — Knowledge Base for Detection Agent

Source: Microsoft Learn, *Deprecated Database Engine features* (SQL Server 2016/2017/2019/2022/2025, view `sql-server-ver17`, updated 2025-11-18). Each entry is self-contained so it stays meaningful when retrieved as an isolated chunk. Scope: only T-SQL syntax detectable in code text (`sys.sql_modules.definition` or DDL). Examples are syntax-reviewed, not executed against a live engine.

---

## text / ntext / image data types (Feature ID 4)
**Deprecated:** `text`, `ntext`, `image` column types
**Substitute:** `varchar(max)`, `nvarchar(max)`, `varbinary(max)`
**Example Deprecated:** `CREATE TABLE t (c TEXT);`
**Example Fixed:** `CREATE TABLE t (c VARCHAR(MAX));`
**FeatureID:** 4
**Deprecated_year:** 2016
**Notes:** Auto-fixable for the type declaration itself (1:1 map: text→varchar(max), ntext→nvarchar(max), image→varbinary(max)). Detect on column definitions in CREATE/ALTER TABLE and on variable/parameter declarations. WARNING: if the column is accessed via WRITETEXT/UPDATETEXT/READTEXT/TEXTPTR (see Feature IDs 5, 6, 114, 115), those statements must be rewritten too — do not change the type in isolation. Column types live in `sys.columns` + `sys.types`, not only in `sys.sql_modules`.

## WRITETEXT / UPDATETEXT (Feature ID 115)
**Deprecated:** `WRITETEXT`, `UPDATETEXT`
**Substitute:** none (use `.WRITE()` on a max-type column)
**Example Deprecated:** `UPDATETEXT t.c @ptr 0 NULL 'x';`
**Example Fixed:** `UPDATE t SET c.WRITE('x', 0, NULL);`
**FeatureID:** 115
**Deprecated_year:** 2016
**Notes:** NOT a 1:1 rewrite. Requires the underlying column to first be migrated to a max type (Feature ID 4). The text-pointer (`@ptr`) workflow disappears entirely. Flag for human review; the agent should not auto-apply because offsets/length semantics differ between UPDATETEXT and `.WRITE()`.

## READTEXT (Feature ID 114)
**Deprecated:** `READTEXT`
**Substitute:** none (use `SUBSTRING` on a max-type column)
**Example Deprecated:** `READTEXT t.c @ptr 0 100;`
**Example Fixed:** `SELECT SUBSTRING(c, 1, 100) FROM t;`
**FeatureID:** 114
**Deprecated_year:** 2016
**Notes:** NOT a 1:1 rewrite. Depends on migrating the column to a max type first. READTEXT offsets are 0-based; SUBSTRING is 1-based — adjust indices when rewriting. Flag for human review.

## TEXTPTR() (Feature ID 5)
**Deprecated:** `TEXTPTR()`
**Substitute:** none (unnecessary with max types)
**Example Deprecated:** `SELECT @ptr = TEXTPTR(c) FROM t;`
**Example Fixed:** not needed once `c` is a `(max)` type
**FeatureID:** 5
**Deprecated_year:** 2016
**Notes:** Text pointers exist only to support the deprecated WRITETEXT/UPDATETEXT/READTEXT workflow. When the column is migrated to a max type, remove TEXTPTR usage entirely rather than replacing it. Detection-only / removal; tie to Feature IDs 4, 114, 115.

## TEXTVALID() (Feature ID 6)
**Deprecated:** `TEXTVALID()`
**Substitute:** none (unnecessary with max types)
**Example Deprecated:** `SELECT TEXTVALID('t.c', TEXTPTR(c)) FROM t;`
**Example Fixed:** not needed once `c` is a `(max)` type
**FeatureID:** 6
**Deprecated_year:** 2016
**Notes:** Same context as TEXTPTR (Feature ID 5): only meaningful in the legacy LOB pointer model. Remove on migration; do not substitute.

## timestamp syntax for rowversion (Feature ID 158)
**Deprecated:** `timestamp` used as the column type keyword
**Substitute:** `rowversion`
**Example Deprecated:** `CREATE TABLE t (v TIMESTAMP);`
**Example Fixed:** `CREATE TABLE t (v ROWVERSION);`
**FeatureID:** 158
**Deprecated_year:** 2016
**Notes:** Auto-fixable 1:1. `timestamp` and `rowversion` are the same data type; only the keyword is deprecated. NOT related to the SQL standard `datetime`/`datetime2` — do not confuse with date/time handling. Detect the `timestamp` keyword only in type position.

## INSERT NULL into a timestamp column (Feature ID 179)
**Deprecated:** explicitly inserting `NULL` into a `timestamp`/`rowversion` column
**Substitute:** use a DEFAULT / omit the column
**Example Deprecated:** `INSERT INTO t (v) VALUES (NULL);`
**Example Fixed:** `INSERT INTO t DEFAULT VALUES;`
**FeatureID:** 179
**Deprecated_year:** 2016
**Notes:** `rowversion` values are auto-generated by the engine; the column should simply be omitted from the INSERT column list. Auto-fixable only if the agent can confirm `v` is the rowversion column and rewrite the column/value lists accordingly. Otherwise flag.

## 'text in row' table option (Feature ID 9)
**Deprecated:** `sp_tableoption ..., 'text in row', ...`
**Substitute:** use `(max)` types (no option needed)
**Example Deprecated:** `EXEC sp_tableoption 't', 'text in row', '1000';`
**Example Fixed:** migrate the column to `VARCHAR(MAX)`; remove the option
**FeatureID:** 9
**Deprecated_year:** 2016
**Notes:** Tied to legacy LOB storage. The fix is to remove the option and migrate to max types (Feature ID 4), not to translate the option. Detection + removal.

## NOLOCK / READUNCOMMITTED in UPDATE or DELETE (Feature ID 1)
**Deprecated:** `NOLOCK`/`READUNCOMMITTED` hint in the target of `UPDATE`/`DELETE`
**Substitute:** remove the hint
**Example Deprecated:** `DELETE FROM t WITH (NOLOCK) WHERE id = 1;`
**Example Fixed:** `DELETE FROM t WHERE id = 1;`
**FeatureID:** 1
**Deprecated_year:** 2016
**Notes:** CONTEXT-SENSITIVE — only deprecated when applied to the table being modified in UPDATE/DELETE. `NOLOCK` in a SELECT (or in a read-only joined table) is NOT covered by this rule. False-positive risk is high with naive text matching; detect on the AST node for the update/delete target table reference specifically. Auto-fixable (removal), but removal changes locking behavior — safe semantically for the modified table.

## Table hints without WITH (Feature ID 8)
**Deprecated:** table hint specified without the `WITH` keyword
**Substitute:** wrap hints in `WITH (...)`
**Example Deprecated:** `SELECT * FROM t (NOLOCK);`
**Example Fixed:** `SELECT * FROM t WITH (NOLOCK);`
**FeatureID:** 8
**Deprecated_year:** 2016
**Notes:** Auto-fixable 1:1. Distinguish a hint in parentheses from a derived-table/subquery parenthesis or a function call — use the AST (hint clause) rather than regex. Common in legacy code as `(NOLOCK)`.

## HOLDLOCK without parentheses (Feature ID 167)
**Deprecated:** `HOLDLOCK` applied without parentheses
**Substitute:** `WITH (HOLDLOCK)`
**Example Deprecated:** `SELECT * FROM t HOLDLOCK;`
**Example Fixed:** `SELECT * FROM t WITH (HOLDLOCK);`
**FeatureID:** 167
**Deprecated_year:** 2016
**Notes:** HIGH PRIORITY — Microsoft lists this in the "deprecated in the next version" group, closer to removal. Auto-fixable 1:1.

## Multiple table hints separated by spaces (Feature ID 168)
**Deprecated:** several hints separated by spaces instead of commas
**Substitute:** comma-separate the hints
**Example Deprecated:** `SELECT * FROM t WITH (NOLOCK ROWLOCK);`
**Example Fixed:** `SELECT * FROM t WITH (NOLOCK, ROWLOCK);`
**FeatureID:** 168
**Deprecated_year:** 2016
**Notes:** Auto-fixable 1:1 (insert commas between hint tokens inside the WITH list). Detect inside the table-hint group in the AST.

## Indirect hints on a multi-statement TVF via a view (Feature ID 7)
**Deprecated:** applying a table hint to a view that wraps a multi-statement table-valued function
**Substitute:** none
**Example Deprecated:** hint applied to a view encapsulating a multi-statement TVF
**Example Fixed:** remove the hint / restructure
**FeatureID:** 7
**Deprecated_year:** 2016
**Notes:** DETECTION-ONLY. The official docs give no precise syntax pattern. Requires resolving the view's definition and the function type — hard to detect statically without object metadata. Flag for human review; do not attempt auto-fix.

## INSERT_HINTS (Feature ID 34)

**Substitute:** —
**Example Deprecated:** — (source does not specify a concrete syntax)
**Example Fixed:** —

## GROUP BY ALL (Feature ID 169)
**Deprecated:** `GROUP BY ALL`
**Substitute:** `UNION` or a derived table (case-by-case)
**Example Deprecated:** `SELECT c, COUNT(*) FROM t WHERE x=1 GROUP BY ALL c;`
**Example Fixed:** rewrite case-by-case with `UNION`/derived table
**FeatureID:** 169
**Deprecated_year:** 2016
**Notes:** NOT a 1:1 rewrite. `GROUP BY ALL` re-introduces groups that the `WHERE` clause filtered out (returning them with a zero/NULL aggregate). Dropping `ALL` silently changes results. The agent must flag this and propose a tested UNION-based rewrite; never auto-apply.

## Indexed-view aggregate select list without COUNT_BIG(*) (compat 90) (Feature ID 2)
**Deprecated:** aggregate indexed view missing `COUNT_BIG(*)` under compatibility level 90
**Substitute:** add `COUNT_BIG(*)`
**Example Deprecated:** `CREATE VIEW v WITH SCHEMABINDING AS SELECT c, SUM(x) AS s FROM dbo.t GROUP BY c;`
**Example Fixed:** `CREATE VIEW v WITH SCHEMABINDING AS SELECT c, SUM(x) AS s, COUNT_BIG(*) AS cnt FROM dbo.t GROUP BY c;`
**FeatureID:** 2
**Deprecated_year:** 2016
**Notes:** Applies to SCHEMABINDING views that will be indexed and contain aggregates. Adding `COUNT_BIG(*)` is required for the indexed view. Semi-auto-fixable: appending `COUNT_BIG(*)` is safe, but verify the view is intended to be indexed before changing its shape.

## :: function call syntax (Feature ID 166)
**Deprecated:** `::function()` call form
**Substitute:** `SELECT ... FROM sys.function()`
**Example Deprecated:** `SELECT * FROM ::fn_virtualfilestats(2,1);`
**Example Fixed:** `SELECT * FROM sys.fn_virtualfilestats(2,1);`
**FeatureID:** 166
**Deprecated_year:** 2016
**Notes:** Auto-fixable 1:1: replace the `::` prefix with the `sys.` schema prefix. Applies to system table-valued functions invoked with the legacy `::` operator.

## Three- or four-part column references (Feature ID 3)
**Deprecated:** column names qualified with 3 or 4 parts
**Substitute:** two-part name (table.column)
**Example Deprecated:** `SELECT mydb.dbo.t.col FROM mydb.dbo.t;`
**Example Fixed:** `SELECT t.col FROM mydb.dbo.t AS t;`
**FeatureID:** 3
**Deprecated_year:** 2016
**Notes:** Deprecation is about the COLUMN reference, not the table reference (the table may still be 3-part). Fix by introducing/using a table alias and qualifying the column with at most two parts. Auto-fixable but requires alias resolution; verify alias uniqueness.

## Quoted string as a column alias (Feature ID 184)
**Deprecated:** `'alias' = expression` form
**Substitute:** `expression AS alias`
**Example Deprecated:** `SELECT 'Total' = SUM(x) FROM t;`
**Example Fixed:** `SELECT SUM(x) AS Total FROM t;`
**FeatureID:** 184
**Deprecated_year:** 2016
**Notes:** Auto-fixable 1:1. Do not confuse with a legitimate string literal in the select list (`SELECT 'Total'`) — the deprecated form has `= expression` after the quoted string. Use the AST select-element node to disambiguate.

## DROP INDEX with two-part name (Feature ID 163)
**Deprecated:** `DROP INDEX table.index`
**Substitute:** `DROP INDEX index ON table`
**Example Deprecated:** `DROP INDEX t.ix;`
**Example Fixed:** `DROP INDEX ix ON t;`
**FeatureID:** 163
**Deprecated_year:** 2016
**Notes:** Auto-fixable 1:1: reverse the order and insert `ON`. The old form is `tableOrViewName.indexName`; the new form is `indexName ON tableOrViewName`.

## Numbered procedures (Feature ID 160)
**Deprecated:** numbered stored procedures (`proc;N`)
**Substitute:** none (use separate procedures)
**Example Deprecated:** `CREATE PROCEDURE p;1 AS ...`
**Example Fixed:** `CREATE PROCEDURE p1 AS ...` (distinct procedures)
**FeatureID:** 160
**Deprecated_year:** 2016
**Notes:** NOT a mechanical rewrite. The `;N` grouping semantics (drop all at once) are lost when splitting into separate procedures. Flag for human review; also check `sys.numbered_procedures` (Feature ID 148/149).

## ROWGUIDCOL as a column name in DML (Feature ID 182)
**Deprecated:** `ROWGUIDCOL` used as a pseudo-column in DML
**Substitute:** `$rowguid`
**Example Deprecated:** `SELECT ROWGUIDCOL FROM t;`
**Example Fixed:** `SELECT $rowguid FROM t;`
**FeatureID:** 182
**Deprecated_year:** 2016
**Notes:** Auto-fixable 1:1. Distinguish the DML pseudo-column usage from the `ROWGUIDCOL` property in a column definition (`ADD c uniqueidentifier ROWGUIDCOL`), which is NOT deprecated.

## IDENTITYCOL as a column name in DML (Feature ID 183)

**Example Fixed:** `SELECT $identity FROM t;`

## '#' / '##' as temporary object names (Feature ID 185)
**Deprecated:** naming a temp table/proc exactly `#` or `##`
**Substitute:** add at least one more character
**Example Deprecated:** `CREATE TABLE ## (c INT);`
**Example Fixed:** `CREATE TABLE ##tmp (c INT);`
**FeatureID:** 185
**Deprecated_year:** 2016
**Notes:** Auto-fixable but renaming a temp object requires updating every reference to it within the same scope/batch. Verify all usages before applying. Rare in practice.

## '@' / '@@' as T-SQL identifiers (Feature ID 186)
**Deprecated:** a variable/identifier named exactly `@` or `@@`
**Substitute:** use a normal identifier name
**Example Deprecated:** `DECLARE @ INT;`
**Example Fixed:** `DECLARE @x INT;`
**FeatureID:** 186
**Deprecated_year:** 2016
**Notes:** Auto-fixable but, like temp-object renaming, requires renaming every reference consistently. Do not confuse `@@`-named identifiers with built-in global functions like `@@ROWCOUNT` (those are not user identifiers and are not deprecated).

## DEFAULT keyword as a default value (Feature ID 187)
**Deprecated:** using the word `DEFAULT` as a default value
**Substitute:** use an explicit value
**Example Deprecated:** uncommon legacy form (source gives no concrete example)
**Example Fixed:** use an explicit default value
**FeatureID:** 187
**Deprecated_year:** 2016
**Notes:** DETECTION-ONLY and UNVERIFIED — the official docs provide no example. Do NOT confuse with the standard, valid uses of `DEFAULT` (e.g. `INSERT ... VALUES (DEFAULT)` or `DEFAULT` constraints), which are fully supported. Flag for manual verification only.

## Statement not terminated by semicolon (no Feature ID)
**Deprecated:** omitting the statement terminator `;`
**Substitute:** terminate statements with `;`
**Example Deprecated:** `SELECT 1`
**Example Fixed:** `SELECT 1;`
**FeatureID:** (none assigned)
**Deprecated_year:** 2016
**Notes:** Style/forward-compatibility deprecation; some constructs (e.g. before `WITH`/CTE, `THROW`, `MERGE`) already REQUIRE a preceding `;`. Auto-fixable but very noisy — recommend treating as low-severity/lint rather than a hard finding. Many statements lack terminators in legacy code.

## Options without parentheses in CREATE/ALTER TABLE or CREATE INDEX (Feature ID 33)
**Deprecated:** WITH-options listed without parentheses
**Substitute:** wrap options in parentheses
**Example Deprecated:** `CREATE INDEX ix ON t(c) WITH FILLFACTOR = 80;`
**Example Fixed:** `CREATE INDEX ix ON t(c) WITH (FILLFACTOR = 80);`
**FeatureID:** 33
**Deprecated_year:** 2016
**Notes:** Auto-fixable 1:1: wrap the comma-separated option list after `WITH` in parentheses. Applies to legacy index/table option syntax.

## ALTER DATABASE MODIFY FILEGROUP READONLY / READWRITE (Feature ID 195 / 196)
**Deprecated:** `READONLY` / `READWRITE` (single word)
**Substitute:** `READ_ONLY` / `READ_WRITE`
**Example Deprecated:** `ALTER DATABASE db MODIFY FILEGROUP fg READONLY;`
**Example Fixed:** `ALTER DATABASE db MODIFY FILEGROUP fg READ_ONLY;`
**FeatureID:** 195 / 196
**Deprecated_year:** 2016
**Notes:** Auto-fixable 1:1: insert the underscore. 195 = READONLY→READ_ONLY, 196 = READWRITE→READ_WRITE. Only in the MODIFY FILEGROUP clause.

## DBCC DBREINDEX (Feature ID 11)
**Deprecated:** `DBCC DBREINDEX`
**Substitute:** `ALTER INDEX ... REBUILD`
**Example Deprecated:** `DBCC DBREINDEX('t', 'ix', 80);`
**Example Fixed:** `ALTER INDEX ix ON t REBUILD WITH (FILLFACTOR = 80);`
**FeatureID:** 11
**Deprecated_year:** 2016
**Notes:** Auto-fixable with care. Map the fillfactor argument to `WITH (FILLFACTOR = n)`. If the index name argument is empty/omitted, DBREINDEX rebuilds ALL indexes → translate to `ALTER INDEX ALL ON t REBUILD`.

## DBCC INDEXDEFRAG (Feature ID 18)

**Substitute:** `ALTER INDEX ... REORGANIZE`
**Example Deprecated:** `DBCC INDEXDEFRAG(db, t, ix);`
**Example Fixed:** `ALTER INDEX ix ON t REORGANIZE;`
**FeatureID:** 18
**Deprecated_year:** 2016
**Notes:** Auto-fixable. The leading database argument is dropped (REORGANIZE runs in the current database context). Map "all indexes" form to `ALTER INDEX ALL ON t REORGANIZE`.

## SET ROWCOUNT for INSERT/UPDATE/DELETE (Feature ID 109)
**Deprecated:** `SET ROWCOUNT n` to limit affected DML rows
**Substitute:** `TOP (n)`
**Example Deprecated:** `SET ROWCOUNT 100; DELETE FROM t WHERE x=1;`
**Example Fixed:** `DELETE TOP (100) FROM t WHERE x=1;`
**FeatureID:** 109
**Deprecated_year:** 2016
**Notes:** Deprecation applies specifically to SET ROWCOUNT's effect on INSERT/UPDATE/DELETE (it no longer affects DML in newer behavior). Rewrite with `TOP (n)`. Remember to also remove the trailing `SET ROWCOUNT 0;` reset if present. SET ROWCOUNT on SELECT is unaffected. Auto-fixable with batch-level analysis.

## SET ANSI_NULLS OFF (Feature ID 111)
**Deprecated:** `SET ANSI_NULLS OFF`
**Substitute:** always `ON`
**Example Deprecated:** `SET ANSI_NULLS OFF;`
**Example Fixed:** `SET ANSI_NULLS ON;`
**FeatureID:** 111
**Deprecated_year:** 2016
**Notes:** WARNING: not a cosmetic change. Code that relies on `= NULL` matching NULLs (the OFF behavior) will break when set to ON — those comparisons must be rewritten to `IS NULL`. Flag for human review; detect OFF specifically, and scan the affected scope for `= NULL` / `<> NULL` predicates.

## SET ANSI_PADDING OFF (Feature ID 113)
**Deprecated:** `SET ANSI_PADDING OFF`
**Substitute:** always `ON`
**Example Deprecated:** `SET ANSI_PADDING OFF;`
**Example Fixed:** `SET ANSI_PADDING ON;`
**FeatureID:** 113
**Deprecated_year:** 2016
**Notes:** Affects trailing-space/zero handling for char/varchar/binary columns. Changing OFF→ON can alter how trailing blanks are stored. Flag for review where column storage behavior matters; safe in most modern code that already assumes ON.

## SET CONCAT_NULL_YIELDS_NULL OFF (Feature ID 112)
**Deprecated:** `SET CONCAT_NULL_YIELDS_NULL OFF`
**Substitute:** always `ON`

**FeatureID:** 112
